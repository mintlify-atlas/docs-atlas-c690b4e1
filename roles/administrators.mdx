Administrators have the highest level of access in the SkillUp platform. They are responsible for approving teacher requests, monitoring platform statistics, and managing overall system health.

### Statistics functions

The dashboard uses dedicated functions to calculate metrics (`pages/admin.php:63-90`):

```php
function getTotalUsers() {
    global $db;
    $sql = "SELECT COUNT(*) as total FROM Utilisateurs";
    $request = $db->prepare($sql);
    $request->execute();
    $result = $request->fetch(PDO::FETCH_ASSOC);
    return $result['total'];
}

function getNewUsers($days) {
    global $db;
    $sql = "SELECT COUNT(*) as total FROM Utilisateurs 
            WHERE date_creation >= NOW() - INTERVAL :days DAY";
    $request = $db->prepare($sql);
    $request->bindParam(':days', $days, PDO::PARAM_INT);
    $request->execute();
    $result = $request->fetch(PDO::FETCH_ASSOC);
    return $result['total'];
}

function getNewCourses($days) {
    global $db;
    $sql = "SELECT COUNT(*) as total FROM Cours 
            WHERE date_creation >= NOW() - INTERVAL :days DAY";
    $request = $db->prepare($sql);
    $request->bindParam(':days', $days, PDO::PARAM_INT);
    $request->execute();
    $result = $request->fetch(PDO::FETCH_ASSOC);
    return $result['total'];
}
```

<Accordion title="How statistics are calculated">
  The statistics use SQL aggregate functions with time-based filtering:
  
  - **Total users**: Simple `COUNT(*)` on the `Utilisateurs` table
  - **Time-based metrics**: Use `NOW() - INTERVAL :days DAY` to filter records
  - **Real-time data**: Statistics are calculated on each page load for accuracy
</Accordion>

## Managing teacher requests

One of the primary responsibilities of administrators is reviewing and approving teacher applications.

### Teacher request workflow

<Steps>
  <Step title="User submits request">
    Students submit teacher requests through their profile with a presentation text
  </Step>
  <Step title="Request appears in admin dashboard">
    The request is queued in the admin dashboard with a unique ID
  </Step>
  <Step title="Administrator reviews">
    Admin reviews the user's presentation and decides to approve or reject
  </Step>
  <Step title="Action taken">
    Admin clicks "Accepter" or "Refuser" to process the request
  </Step>
</Steps>

### Viewing pending requests

The admin dashboard displays all pending teacher requests (`pages/admin.php:148-183`):


Each request shows:

- **Request ID**: Unique identifier (displayed as "DEM-{id}")
- **Presentation**: User's explanation of why they want to become a teacher
- **Action buttons**: Accept or refuse the request

<Note>
  If there are no pending requests, the dashboard displays "Aucune demande en cours" to indicate the queue is empty.
</Note>

## Approving teachers

When an administrator approves a teacher request, several actions occur (`pages/admin.php:36-61`):

### Approval process

```php
function AddTeacher($user) {
    global $db;
    
    // 1. Update user role to 'professeur'
    $sql = "UPDATE Utilisateurs SET role = 'professeur' 
            WHERE id = :id";
    $request = $db->prepare($sql);
    $request->bindParam(':id', $user);
    $request->execute();
    
    // 2. Delete the teacher request
    DeleteRequest($user);
    
    // 3. Generate API token for the new teacher
    $token = generateToken();
    $currentDate = date('Y-m-d H:i:s');
    $sql = "INSERT INTO KeyTable (actif, token, date_creation) 
            VALUES (true, :token, :date)";
    $request = $db->prepare($sql);
    $request->execute();
    $keyId = $db->lastInsertId();
    
    // 4. Link API key to user account
    $sql = 'UPDATE Utilisateurs SET key_id = :kid WHERE id = :id';
    $request = $db->prepare($sql);
    $request->execute();
}
```

## Rejecting teacher requests

Administrators can reject teacher requests if they don't meet criteria (`pages/admin.php:29-35`):

```php
function DeleteRequest($user) {
    global $db;
    $sql = "DELETE FROM DemandeProf WHERE id_utilisateur = :id";
    $request = $db->prepare($sql);
    $request->bindParam(':id', $user);
    $request->execute();
}
```

Rejecting a request:

- Removes the entry from the `DemandeProf` table
- Does not notify the user (they can submit a new request)
- Does not change the user's role or permissions

<Note>
  Users whose requests are rejected can submit new teacher requests at any time through their profile page.
</Note>

## Request processing

The admin dashboard handles both approval and rejection through a single form (`pages/admin.php:92-100`):

```php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    if (isset($_POST['user']) && isset($_POST['action'])) {
        if ($_POST['action'] == "1") {
            // Approve the request
            AddTeacher($_POST['user']);
        } else {
            // Reject the request
            DeleteRequest($_POST['user']);
        }
    }
}
```

The `action` parameter determines the outcome:

- `action="1"`: Approve and grant teacher role
- `action="0"`: Reject and delete request

## Security features

Administrators benefit from the same security measures as other users, plus additional protections:

### Session management

Admin sessions are validated using the same `IsConnected()` function (`include/sessionManager.php:4-35`):

```php
function IsConnected(): bool {
    // Delete expired sessions on each check
    $now = new DateTime();
    $sql = "DELETE FROM sessions WHERE expires_at < :now";
    $stmt->execute();

    if (!isset($_COOKIE['user_token'])) {

    // Check session hasn't expired
    $expires_at = new DateTime($result['expires_at']);

```

### API rate limiting

While administrators don't directly use the API, the system implements rate limiting to protect all endpoints (`include/secureCheck.php:4-25`):

```php
function LogsCheck() {
    $ip = GetIP();
    $dateH_check = (new DateTime('-30 minutes'))->format('Y-m-d H:i:s');
    
    // Check for failed attempts in last 30 minutes
    $sql = 'SELECT COUNT(ApiLogs.id) AS nb_denied 
            FROM ApiLogs 
            WHERE date_heure > :dateheure 
              AND succes = FALSE 
              AND ip = :ip';
    
    $result = $request->fetch(PDO::FETCH_ASSOC);
    
    // Block if more than 10 failed attempts
    if ($result['nb_denied'] < 10) {
        return true;
    } else {
        return false;
    }
}
```

<Warning>
  IP addresses with more than 10 failed API requests in 30 minutes are automatically blocked. This protects against brute force attacks and abuse.
</Warning>

### User management considerations

When approving teachers, consider:

- **Presentation quality**: Does the user demonstrate knowledge and commitment?
- **Account age**: Is this a new account or an established user?
- **Platform growth**: Are you maintaining a good student-to-teacher ratio?
- **Content needs**: What topics or categories need more teachers?

<Accordion title="What happens when a teacher account is compromised?">
  If a teacher's account is compromised:
  
  1. The teacher can regenerate their API key from the API page
  2. Administrators don't have direct tools to revoke teacher status
  3. Consider implementing a manual role downgrade if needed
  4. Monitor course creation and modification logs for suspicious activity
</Accordion>
