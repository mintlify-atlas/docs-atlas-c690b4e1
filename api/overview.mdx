The SkillUp API provides a RESTful interface for teachers to programmatically manage course content, chapters, and images. This API enables integration with external tools and automation of course creation workflows.

## Who can access the API

<Warning>
The SkillUp API is exclusively available to users with the **teacher** (professeur) role. Students cannot access the API.
</Warning>

To use the API, you must:
- Have a verified teacher account on the SkillUp platform
- Possess a valid API key linked to your account
- Include your API key with every request

## Base URL

All API endpoints are accessed via HTTPS:

```text
https://skillup.great-site.net/api/
```

## Response format

All API responses are returned in JSON format with appropriate HTTP status codes.

### Success response

Successful requests return HTTP `200` with a JSON object:

```json
{
  "success": true,
  "content": "# Chapter content here..."
}
```

### Error response

Failed requests return HTTP `401` with an error message:

```json
{
  "error": "Invalid or expired token"
}
```

## Error handling

The API implements several error handling mechanisms:

<Steps>
  <Step title="Token validation">
    Every request validates the API token against the `KeyTable` database. Invalid tokens return a `401` error.
  </Step>
  
  <Step title="Rate limiting">
    The API tracks failed requests by IP address. More than 10 failed requests within 30 minutes will temporarily block access.
  </Step>
  
  <Step title="Request logging">
    All API requests (successful and failed) are logged in the `ApiLogs` table with timestamp and IP address.
  </Step>
</Steps>

### Common error messages

| Error Message | Cause | Solution |
|--------------|-------|----------|
| `Invalid or expired token` | Token not found in database | Regenerate your API key |
| `Syntax error in request` | Missing required parameters | Check parameter names and values |
| `Can't found this chapter` | Chapter ID doesn't exist | Verify the chapter and course IDs |
| `Unknown categorie` | Invalid category name | Use exact category names from documentation |

## Request method

<Info>
All API endpoints accept **POST** requests with `application/x-www-form-urlencoded` content type.
</Info>
