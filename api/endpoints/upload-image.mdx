---
title: Upload image
description: Upload a base64-encoded image to a course chapter
---

## Endpoint

```http
POST /api/UploadImage.php
```

Upload an image in base64 markdown format to a specific chapter within a course. The image is decoded and saved as a PNG file in the chapter's images directory.

## Authentication

This endpoint requires a valid authentication token and passes a logs check. The token must be included in the request body.

<Warning>
  Invalid or expired tokens will result in a 401 Unauthorized response. This endpoint also requires passing the LogsCheck() validation.
</Warning>

## Request parameters

<ParamField body="token" type="string" required>
  Authentication token obtained from login
</ParamField>

<ParamField body="image" type="string" required>
  Base64-encoded image in markdown format. Must match the pattern:
  ```markdown
  ![alt text](data:image/png;base64,BASE64_DATA_HERE)
  ```
</ParamField>

<ParamField body="cours" type="string" required>
  The course ID where the image will be stored
</ParamField>

<ParamField body="chapter" type="string" required>
  The chapter ID where the image will be associated
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates whether the image was uploaded successfully
</ResponseField>

<ResponseField name="file" type="string">
  The file path where the image was saved
</ResponseField>

<ResponseField name="error" type="string">
  Error message if the request failed
</ResponseField>

## Response codes

| Code | Description |
|------|-------------|
| 200  | Image uploaded successfully |
| 401  | Invalid token, missing parameters, invalid format, or upload error |

## Image format validation

The endpoint validates the image using a regex pattern:

```regex
/^!\[.*\]\(data:image\/([a-zA-Z]*);base64,([A-Za-z0-9+\/=_-]+)\)$/
```

This ensures the image is provided in the correct markdown base64 format.

## Error messages

- `"Invalid token"` - Authentication token is invalid or has expired
- `"Missing parameters"` - One or more required parameters are missing
- `"Format base64 markdown invalide"` - The image format doesn't match the expected base64 markdown pattern
- `"Error can't save image, check cours and chapter values !"` - Unable to save the image file to disk
- `"Can't create images dir in your cours"` - Unable to create the images directory

## File structure

Images are stored in the following directory structure:

```text
../md_files/
└── {cours}/
    └── images/
        └── {chapter}/
            └── {image_id}.png
```

The directory structure is created automatically if it doesn't exist (with permissions 0777).

## Database operations

When an image is uploaded, a record is inserted into the `Images` table:

```sql
INSERT INTO Images (Images.id_chapitre) VALUES (:idc);
```

The last inserted ID is used as the image filename.

  <Tab title="PHP">
    ```php
    <?php
    $url = 'https://yourdomain.com/api/UploadImage.php';
    
    // Read image and convert to base64
    $imageData = file_get_contents('path/to/image.png');
    $base64 = base64_encode($imageData);
    $markdownImage = "![Uploaded Image](data:image/png;base64,{$base64})";

    $data = [
        'token' => 'your_auth_token_here',
        'cours' => '123',
        'chapter' => '456',
        'image' => $markdownImage
    ];

    $options = [
        'http' => [
            'header'  => "Content-type: application/x-www-form-urlencoded\r\n",
            'method'  => 'POST',
            'content' => http_build_query($data)
        ]
    ];

    $context  = stream_context_create($options);
    $result = file_get_contents($url, false, $context);
    $response = json_decode($result, true);

    if ($response['success']) {
        echo "Image uploaded to: " . $response['file'];
    } else {
        echo "Error: " . $response['error'];
    }
    ?>
    ```
  </Tab>
  
  <Tab title="JavaScript">
    ```javascript
    // Convert file to base64
    function fileToBase64(file) {
      return new Promise((resolve, reject) => {
        const reader = new FileReader();
        reader.readAsDataURL(file);
        reader.onload = () => {
          const base64 = reader.result.split(',')[1];
          const markdown = `![image](data:image/png;base64,${base64})`;
          resolve(markdown);
        };
        reader.onerror = error => reject(error);
      });
    }

    // Upload image
    const fileInput = document.getElementById('imageInput');
    const file = fileInput.files[0];

    fileToBase64(file).then(markdownImage => {
      const formData = new URLSearchParams();
      formData.append('token', 'your_auth_token_here');
      formData.append('cours', '123');
      formData.append('chapter', '456');
      formData.append('image', markdownImage);

      fetch('https://yourdomain.com/api/UploadImage.php', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/x-www-form-urlencoded',
        },
        body: formData
      })
      .then(response => response.json())
      .then(data => {
        if (data.success) {
          console.log('Image uploaded to:', data.file);
        } else {
          console.error('Error:', data.error);
        }
      });
    });
    ```
  </Tab>

<Note>
  Only PNG format is supported. The image is always saved with a `.png` extension regardless of the original format specified in the base64 data URI.
</Note>

<Warning>
  Large base64 strings may exceed POST size limits. Consider adjusting your PHP configuration (`post_max_size` and `upload_max_filesize`) if you need to upload large images.
</Warning>
