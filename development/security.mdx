## Overview

SkillUp implements multiple layers of security to protect user data, prevent common web vulnerabilities, and ensure safe operation. The platform follows industry best practices for PHP web application security.

<Warning>
Security is a critical aspect of the platform. All developers must understand and follow these security practices when contributing code.
</Warning>

## Authentication system

### Password hashing

All user passwords are hashed using PHP's built-in `password_hash()` function with the bcrypt algorithm.

**Implementation** (`/pages/connection.php:28`)

```php
// During registration
function registration() {
    global $db, $message;
    $password = $_POST['password'];
    $confirmedPassword = $_POST['cPassword'];
    
    if ($password == $confirmedPassword) {
        $hashedPassword = password_hash($password, PASSWORD_DEFAULT);
        $sql = "INSERT INTO Utilisateurs (nom, prenom, e_mail, mot_de_passe, date_creation) 
                VALUES (:nom, :prenom, :email, :mot_de_passe, :date_creation)";
        $request = $db->prepare($sql);
        $request->bindParam(':mot_de_passe', $hashedPassword);
        $request->execute();
    }
}
```

**Password verification** (`/pages/connection.php:73`)

```php
function connection() {
    global $db, $message;
    $email = $_POST['email'];
    $password = $_POST['password'];
    
    $sql = "SELECT id, nom, prenom, mot_de_passe FROM Utilisateurs WHERE e_mail = :email";
    $request = $db->prepare($sql);
    $request->bindParam(':email', $email);
    $request->execute();
    $result = $request->fetch(PDO::FETCH_ASSOC);

    if (($result) && password_verify($password, $result['mot_de_passe'])) {
        // Password is correct - create session
    } else {
        $message = "<p>Email ou mot de passe incorrect !</p>";
    }
}
```

<Tabs>
  <Tab title="Algorithm">
    `PASSWORD_DEFAULT` currently uses bcrypt, which:
    - Automatically generates a salt
    - Uses a cost factor of 10 by default
    - Produces a 60-character hash
    - Is resistant to brute-force attacks
  </Tab>
  
  <Tab title="Storage">
    Hashed passwords are stored in the `Utilisateurs.mot_de_passe` field:
    - Type: `varchar(255)`
    - Example: `$2y$10$LscVQZXU37FxAfV4i75f.e5LDTzZE7Q7bEMJHXk58sBnNtW/EjyCq`
    - Never stored in plain text
    - Never logged or displayed
  </Tab>
  
  <Tab title="Best practices">
    - Minimum password length: 8 characters (enforced in HTML)
    - Maximum length: 255 characters
    - Password confirmation required during registration
    - No password hints or recovery (only reset)
    - Passwords never transmitted unencrypted
  </Tab>
</Tabs>

## SQL injection prevention

### Prepared statements

All database queries use PDO prepared statements with parameter binding to prevent SQL injection attacks.

**Secure query example** (`/include/secureCheck.php:60-72`)

```php
function CheckToken($token) {
    $db = connectDB();
    $sql = 'SELECT Utilisateurs.id FROM Utilisateurs 
            INNER JOIN KeyTable ON KeyTable.key_id = Utilisateurs.key_id 
            WHERE KeyTable.token = :token';
    $request = $db->prepare($sql);
    $request->bindParam(':token', $token);
    $request->execute();
    $user = $request->fetch(PDO::FETCH_ASSOC);
    
    if ($user) {
        return true;
    } else {
        return false;
    }
}
```

<Accordion title="Why prepared statements?">
Prepared statements separate SQL code from data:

**Vulnerable code (DON'T DO THIS):**
```php
// VULNERABLE TO SQL INJECTION!
$sql = "SELECT * FROM Utilisateurs WHERE e_mail = '" . $_POST['email'] . "'";
$result = $db->query($sql);
```

An attacker could input: `' OR '1'='1` to bypass authentication.

**Secure code:**
```php
// SAFE - Uses prepared statement
$sql = "SELECT * FROM Utilisateurs WHERE e_mail = :email";
$request = $db->prepare($sql);
$request->bindParam(':email', $_POST['email']);
$request->execute();
```

The parameter is treated as data, not executable SQL.
</Accordion>

### PDO configuration

The database connection is configured with secure settings (`/include/connectdb.php:7-10`):

```php
$options = [
    PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_EMULATE_PREPARES => false
];
```

<Info>
`PDO::ATTR_EMULATE_PREPARES => false` ensures the database server handles parameter binding, providing stronger protection against SQL injection.
</Info>

## Session security

### Token-based sessions

SkillUp uses secure token-based session management instead of PHP's default session handling.

**Session creation** (`/pages/connection.php:74-86`)

```php
if (($result) && password_verify($password, $result['mot_de_passe'])) {
    // Generate unique session token
    $token_seed = $result['id'] . (new DateTime())->format('Y-m-d H:i:s');
    $token = password_hash($token_seed, PASSWORD_DEFAULT);

    // Set HTTP-only cookie (not accessible via JavaScript)
    setcookie("user_token", $token, time() + 86400, "/");
    
    // Store session in database
    $sql_session = 'INSERT INTO sessions (user_id, token, expires_at) 
                    VALUES (:id, :token, :expiresDate)';
    $request_session = $db->prepare($sql_session);
    $request_session->bindParam(':id', $result['id']);
    $request_session->bindParam(':token', $token);
    $expiresDate = new DateTime();
    $expiresDate->modify('+1 day'); // 24-hour sessions
    $expiresDateFormatted = $expiresDate->format('Y-m-d H:i:s');
    $request_session->bindParam(':expiresDate', $expiresDateFormatted);
    $request_session->execute();
}
```

### Session validation

Every protected page checks if the user is authenticated using `IsConnected()` (`/include/sessionManager.php:4-35`):

```php
function IsConnected(): bool {
    $db = connectDB();

    // Auto-delete expired sessions
    $now = new DateTime();
    $nowFormatted = $now->format('Y-m-d H:i:s');
    $sql = "DELETE FROM sessions WHERE expires_at < :now";
    $stmt = $db->prepare($sql);
    $stmt->bindParam(':now', $nowFormatted);
    $stmt->execute();

    // Check if cookie exists
    if (!isset($_COOKIE['user_token'])) {
        return false;
    }
    
    // Validate token in database
    $sql = "SELECT user_id, expires_at FROM sessions WHERE token = :token";
    $stmt = $db->prepare($sql);
    $stmt->bindParam(':token', $_COOKIE['user_token'], PDO::PARAM_STR);
    $stmt->execute();
    $result = $stmt->fetch(PDO::FETCH_ASSOC);

    if (!$result) {
        return false;
    }

    // Check expiration
    $expires_at = new DateTime($result['expires_at']);
    if ($expires_at > $now) {
        return true; // Valid session
    }

    return false; // Expired
}
```

<Accordion title="Session security features">
- **Hashed tokens**: Session tokens are hashed before storage
- **Database-backed**: All sessions stored in database, not files
- **Auto-expiration**: Expired sessions automatically deleted
- **24-hour lifetime**: Sessions expire after 1 day
- **Per-request validation**: Every request checks session validity
- **Server-side state**: Session data never stored in cookies
</Accordion>

## API rate limiting

### Request logging

All API requests are logged to track usage and detect abuse (`/include/secureCheck.php:26-34`):

```php
function AddInLog($state) {
    $db = connectDB();
    $ip = GetIP();
    $sql = 'INSERT INTO ApiLogs (ApiLogs.ip, ApiLogs.succes) VALUES (:ip, :success)';
    $request = $db->prepare($sql);
    $request->bindParam(':ip', $ip);
    $request->bindParam(':success', $state);
    $request->execute();
}
```

### IP-based rate limiting

The `LogsCheck()` function prevents API abuse by limiting failed requests (`/include/secureCheck.php:4-25`):

```php
function LogsCheck() {
    $ip = GetIP();

    if (!empty($ip)) {
        $db = connectDB();
        $dateH_check = (new DateTime('-30 minutes'))->format('Y-m-d H:i:s');
        $sql = 'SELECT COUNT(ApiLogs.id) AS nb_denied FROM ApiLogs 
                WHERE ApiLogs.date_heure > :dateheure 
                AND ApiLogs.succes = FALSE 
                AND ip = :ip';
        $request = $db->prepare($sql);
        $request->bindParam(':dateheure', $dateH_check);
        $request->bindParam(':ip', $ip);
        $request->execute();
        $result = $request->fetch(PDO::FETCH_ASSOC);
        
        if ($result['nb_denied'] < 10) {
            return true; // Allow request
        } else {
            return false; // Block request
        }
    } else {
        return false;
    }
}
```

<Warning>
IPs with 10 or more failed API requests in a 30-minute window are automatically blocked.
</Warning>

### IP detection

The platform correctly identifies user IPs even behind proxies (`/include/secureCheck.php:50-59`):

```php
function GetIP() {
    $ip = $_SERVER['REMOTE_ADDR'];
    
    // Check for proxy headers
    if (!empty($_SERVER['HTTP_CLIENT_IP'])) {
        $ip = $_SERVER['HTTP_CLIENT_IP'];
    } elseif (!empty($_SERVER['HTTP_X_FORWARDED_FOR'])) {
        $ip = $_SERVER['HTTP_X_FORWARDED_FOR'];
    }

    return $ip;
}
```

## API authentication

### Token generation

Professor accounts receive unique API tokens for course creation (`/include/secureCheck.php:35-49`):

```php
function generateToken() {
    $exists = 1;
    while ($exists > 0) {
        $db = connectDB();
        $token = bin2hex(random_bytes(32)); // Generate 64-char hex token
        $sql = 'SELECT COUNT(*) FROM KeyTable WHERE token = :token';
        $request = $db->prepare($sql);
        $request->bindParam(':token', $token);
        $request->execute();
        $exists = $request->fetchColumn();
    }
    
    return $token;
}
```

<Tabs>
  <Tab title="Token format">
    - Generated with `random_bytes(32)` for cryptographic security
    - Converted to hexadecimal (64 characters)
    - Example: `98e56f053717bf17088136a00671218abd59a776404a98d907e69616b917977f`
    - Uniqueness verified before insertion
  </Tab>

## Security best practices

<Note>
When contributing to SkillUp, always:

1. Use prepared statements for ALL database queries
2. Escape ALL output with `htmlspecialchars()`
3. Validate ALL user input (type, length, format)
4. Hash passwords with `password_hash()`
5. Never log or display sensitive data
6. Check `IsConnected()` on protected pages
7. Use `LogsCheck()` on API endpoints
8. Handle errors gracefully without exposing details
</Note>
