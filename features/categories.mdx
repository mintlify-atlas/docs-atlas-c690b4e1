## Overview

The categories system helps organize courses into topic areas, making it easier for students to discover relevant content. Categories are used throughout the platform for filtering, navigation, and course organization.

## Category browsing

Students can explore all available categories from the dedicated categories page.

### Fetch all categories

From `pages/categories.php:9-16`:

```php
$query = "SELECT * FROM categories";

$stmt = $db->prepare($query);
$stmt->execute();

$categories = $stmt->fetchAll(PDO::FETCH_ASSOC);
```

## Category filtering

Categories are used to filter courses on the formations page.

### Category dropdown filter

From `pages/formations.php:11-14`:

```php
$categoriesQuery = "SELECT id, nom FROM Categories ORDER BY nom";
$categoriesStmt = $db->prepare($categoriesQuery);
$categoriesStmt->execute();
$categories = $categoriesStmt->fetchAll(PDO::FETCH_ASSOC);
```

### Filter implementation

```html
<div class="filters">
    <form action="formations.php" method="get">
        <?php if (!empty($recherche)): ?>
            <input type="hidden" name="recherche" 
                   value="<?php echo htmlspecialchars($recherche); ?>">
        <?php endif; ?>
        <select name="categorie" onchange="this.form.submit()">
            <option value="">Toutes les catégories</option>
            <?php foreach ($categories as $cat): ?>
                <option value="<?php echo $cat['id']; ?>" 
                        <?php echo ($categorie == $cat['id']) ? 'selected' : ''; ?>>
                    <?php echo htmlspecialchars($cat['nom']); ?>
                </option>
            <?php endforeach; ?>
        </select>
    </form>
</div>
```

<Note>
  The filter preserves search terms when switching categories, providing a seamless filtering experience.
</Note>

### Update course category

From `pages/addCourse.php:62-70`:

```php
function ChangeName($name, $cat) {
    global $db;
    $sql = "UPDATE Cours SET Cours.nom = :name, Cours.categorie_id = :categorie 
            WHERE Cours.id = :id";
    $request = $db->prepare($sql);
    $request->bindParam(":name", $name);
    $request->bindParam(":categorie", $cat);
    $request->bindParam(":id", $_GET['cours']);
    $request->execute();
}
```

## Database schema

### Categories table

```sql
CREATE TABLE Categories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nom VARCHAR(100) NOT NULL,
    description TEXT
);
```

### Relationship to Cours table

```sql
CREATE TABLE Cours (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nom VARCHAR(255) NOT NULL,
    illustration_url TEXT,
    description TEXT,
    prof_id INT NOT NULL,
    categorie_id INT,
    date_creation DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (prof_id) REFERENCES Utilisateurs(id) ON DELETE CASCADE,
    FOREIGN KEY (categorie_id) REFERENCES Categories(id)
);
```

## Category filtering workflow

<Steps>
  <Step title="User selects category">
    From the categories page, dropdown filter, or footer links.
  </Step>
  
  <Step title="Query with filter">
    The formations page receives the category ID as a GET parameter.
  </Step>
  
  <Step title="Database filtering">
    SQL query filters courses WHERE categorie_id matches the selected category.
  </Step>
  
  <Step title="Display results">
    Only courses in the selected category are shown in the grid.
  </Step>
  
  <Step title="Maintain filter">
    Selected category remains highlighted in the dropdown for user context.
  </Step>
</Steps>

## Handling uncategorized courses

Courses without an assigned category are handled gracefully:

```php
echo htmlspecialchars($row['categorie_nom'] ?? 'Non catégorisé');
```

<Note>
  The `??` null coalescing operator displays "Non catégorisé" when a course has no category assigned.
</Note>

## Best practices

<Accordion title="For professors">
  - Always assign a category to your courses
  - Choose the most specific category that applies
  - Update category if course content changes significantly
  - Use categories to help students find your courses
</Accordion>

<Accordion title="For platform administrators">
  - Keep category names clear and concise
  - Provide descriptive text for each category
  - Avoid creating too many similar categories
  - Monitor which categories are most popular
  - Consider adding new categories based on course submissions
</Accordion>

## Future enhancements

Potential improvements to the category system:

- **Hierarchical categories**: Parent/child category relationships
- **Multiple categories**: Allow courses to belong to multiple categories
- **Category icons**: Visual representation for each category
- **Popular categories**: Sort by number of courses or enrollments
- **Category subscriptions**: Notify users when new courses are added to favorite categories
- **Tag system**: Complement categories with flexible tags

<Warning>
